▼

Vector Databases & Embeddings

~18 hrs
▼
Monday - Tuesday (5 hrs)

Sunday (3 hrs)

  • Write a README explaining each component and your evaluation methodology
  • Push to GitHub as "semantic-search-rag" project
🎯 Week 7 Deliverable

A working RAG pipeline with: (1) document ingestion + chunking, (2) vector DB storage, (3) semantic retrieval, (4) LLM generation, (5) retrieval evaluation metrics (precision@k, recall@k).

💡 QA Leverage

Chunking strategy is test case granularity. Too big = missed edge cases. Too small = lost context. You already know how to balance coverage vs. maintainability in test suites.
